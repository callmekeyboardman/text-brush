# 设置界面迭代方案设计

> ✅ **已实施完成**（2026-07）。本文档为方案设计，实施记录见文末「实施记录」章节。最终代码以 [src/settingTab.ts](src/settingTab.ts) 为准。

## 背景

当前设置页面（`src/settingTab.ts`）把语言选择、颜色、字号全部平铺在一个滚动页面里。随着插件功能增加，页面会越来越拥挤。需要重构为 Tab 导航结构，参考 claudian 插件的实现方式。

---

## 目标结构

```
通用        文字颜色      字体
─────────────────────────────
[ 语言选择 ] [ 颜色列表 ]  [ 字号列表 ]
```

| Tab ID     | 显示名（中/英）  | 当前对应的设置项         |
|------------|----------------|----------------------|
| `general`  | 通用 / General  | 语言选择               |
| `colors`   | 文字颜色 / Text Color | 颜色调色板           |
| `fonts`    | 字体 / Font     | 字号列表               |

新功能接入时，只需在 `TAB_IDS` 数组末尾追加新 ID，再补充对应的渲染方法，无需改动已有 Tab 逻辑。

---

## 数据模型：完全不变

`TextColorSettings`（`src/types.ts`）无需任何改动：

```typescript
// 保持现状，不增不减
export interface TextColorSettings {
    language: LangSetting;
    colors: ColorOption[];
    fontSizes: FontSizeOption[];
}
```

已有用户的 `data.json` 中存储的 `colors`、`fontSizes` 字段在 `loadSettings()` 里直接被读取和合并，重构后行为不变，**不会丢失任何用户数据**。

## settingTab.ts 重构

### 核心改动

1. 新增 `activeTab` 私有状态，记住用户上次选择的 Tab，重新打开设置时保持选中状态。
2. `display()` 方法只负责渲染 Tab 栏和容器骨架，具体内容交给各 `render*Tab()` 方法。
3. 每个功能域对应一个私有方法，互不干扰，方便后续扩展。

### 伪代码结构

```typescript
type TabId = 'general' | 'colors' | 'fonts';

// 后续增加功能时，在此追加新 Tab ID，再补 render 方法即可
const TAB_IDS: TabId[] = ['general', 'colors', 'fonts'];

export class TextColorSettingTab extends PluginSettingTab {
    private activeTab: TabId = 'general';

    display(): void {
        const { containerEl } = this;
        containerEl.empty();
        containerEl.addClass('tb-settings');

        const t = this.plugin.t;

        // 1. 渲染 Tab 栏
        const tabBar = containerEl.createDiv({ cls: 'tb-settings-tabs' });
        const tabButtons = new Map<TabId, HTMLButtonElement>();
        const tabContents = new Map<TabId, HTMLDivElement>();

        for (const id of TAB_IDS) {
            const label = this.getTabLabel(id, t);
            const btn = tabBar.createEl('button', {
                cls: `tb-settings-tab${id === this.activeTab ? ' tb-settings-tab--active' : ''}`,
                text: label,
            });
            btn.addEventListener('click', () => {
                this.activeTab = id;
                for (const tabId of TAB_IDS) {
                    tabButtons.get(tabId)?.toggleClass('tb-settings-tab--active', tabId === id);
                    tabContents.get(tabId)?.toggleClass('tb-settings-tab-content--active', tabId === id);
                }
            });
            tabButtons.set(id, btn);
        }

        // 2. 创建各 Tab 内容容器
        for (const id of TAB_IDS) {
            const content = containerEl.createDiv({
                cls: `tb-settings-tab-content${id === this.activeTab ? ' tb-settings-tab-content--active' : ''}`,
            });
            tabContents.set(id, content);
        }

        // 3. 渲染各 Tab 内容
        this.renderGeneralTab(tabContents.get('general')!, t);
        this.renderColorsTab(tabContents.get('colors')!, t);
        this.renderFontsTab(tabContents.get('fonts')!, t);
    }

    private getTabLabel(id: TabId, t: Translations): string {
        const map: Record<TabId, string> = {
            general: t.settingTabGeneral,
            colors:  t.settingTabColors,
            fonts:   t.settingTabFonts,
        };
        return map[id];
    }

    private renderGeneralTab(container: HTMLElement, t: Translations): void {
        // 语言选择 —— 原 display() 中语言 Setting 的代码移入此处，逻辑不变
    }

    private renderColorsTab(container: HTMLElement, t: Translations): void {
        // 颜色列表 —— 原 display() 中颜色相关 Setting 的代码移入此处，逻辑不变
    }

    private renderFontsTab(container: HTMLElement, t: Translations): void {
        // 字号列表 —— 原 display() 中字号相关 Setting 的代码移入此处，逻辑不变
    }
}
```

## 向后兼容性保障

- **数据结构基本不变**：`TextColorSettings` 仅新增 `activeSettingsTab` 字段（见实施记录），原有 `colors`/`fontSizes`/`language` 字段名和类型不变。老用户的 `data.json` 读取后经 spread 合并，`activeSettingsTab` 缺失时由 `DEFAULT_SETTINGS` 兜底为 `'general'`，自定义颜色和字号不会丢失。
- **`loadSettings()` 不变**：已有的 "没有存颜色则用默认值" 逻辑不受影响。
- **`colorsAreBuiltinDefaults()` 不变**：语言切换时判断是否覆盖颜色名称的逻辑不变。

## 实施记录

方案于 2026-07 实施完成。实施过程中相对原方案有以下调整：

### 调整 1：`activeTab` 改为持久化到 settings

**原方案**：`activeTab` 为实例字段（`private activeTab: TabId = 'general'`），声称"重新打开设置时保持选中状态"。

**问题**：Obsidian 每次打开设置页会重建 `TextColorSettingTab` 实例，实例字段随之丢失，"记住上次 Tab"的设计目标无法达成——每次都回到 `'general'`。

**实施调整**：
- `src/types.ts` 的 `TextColorSettings` 新增 `activeSettingsTab: SettingsTabId` 字段，并导出 `SettingsTabId` 类型
- `src/settingTab.ts` 的 `activeTab` 改为 getter/setter 对，背后读写 `plugin.settings.activeSettingsTab`；getter 用 `TAB_IDS.includes` 校验，防止 data.json 里存了已废弃的 id
- Tab 按钮 click 回调里 setter 之后立即 `saveSettings()` 持久化

老用户 data.json 无此字段时，`loadSettings()` 的默认值合并兜底为 `'general'`，向后兼容。

### 调整 2：Tab 切换不触发重绘

原方案伪代码中 Tab 切换仅切换 class（不调用 `display()`），实施时严格保持了这一点——切换 Tab 只 `toggleClass`，响应即时；只有增删项、切换语言时才 `display()` 重建。这与原方案一致，此处仅作确认。

### 验证

- `npm run typecheck` + `npm run build` 通过
- 手动测试：切换 Tab → 关闭设置页重开 → 确认恢复到上次查看的 Tab；颜色/字号增删改、语言切换均正常
