# 知识卡片-7：多语言（i18n）架构

> **实施状态：已完成 ✅**（2026-07-11）
>
> 将 i18n 层从「单文件内联 2 语言 + 对象访问」重构为「独立 JSON × 10 语言 + `t(key, locale)` 函数访问 + 英文回退 + 编译期 key 校验」。参考了同作者另一插件 `claudian` 的 i18n 实现。`npm run build`（`tsc -noEmit` + esbuild）通过，10 个 locale 文件 key 一致性校验通过（各 42 keys）。

## 1. 背景与目标

### 1.1 重构前

i18n 全部塞在单文件 `src/i18n.ts` 中：

- `TRANSLATIONS: Record<Lang, Translations>` 内联对象，`Lang = 'en' | 'zh'` 仅 2 语言
- 消费方用对象属性访问：`this.t.menuTextColor`、`t.settingTabGeneral`
- `this.t` getter 每次返回整个 `Translations` 对象，设置页 `display()` 把对象层层传递
- 无回退机制：某语言漏 key 直接显示 `undefined`
- 语言检测只区分 `zh` / `en` 两档——Obsidian 设日语界面时 `auto` 也显示英文

### 1.2 重构动机

计划扩展到 10 种语言。继续用单文件内联会让 `i18n.ts` 膨胀到难以维护；缺键回退、key 一致性校验等问题在 10 语言下会放大。

### 1.3 目标

- 翻译数据拆为独立 JSON，每种语言一个文件，`en.json` 为源真理
- 改用 `t(key, locale)` 函数调用，dot-path 命名空间
- 缺键自动回退英文，仍缺返回 key 字符串
- 编译期校验 key（写错 key 直接报 TS 错误）
- 保留 `auto` 跟随 Obsidian，但映射覆盖全部 10 种语言
- 中文采用标准 locale 码（`zh-CN` / `zh-TW`），向后兼容旧 `zh` 值

参考实现：`S:\claudian\src\i18n\`（10 语言、`t(key)` 函数、JSON 拆分、`TranslationKey` 类型推导）。

## 2. 设计决策

### 2.1 API 模式：`t(key, locale)` 而非模块级单例

claudian 用模块级 `currentLocale` 单例 + `setLocale()` + `t(key)`。本插件**没有照搬**，而是采用 `t(key, locale, params?)` ——locale 由调用方传入。

**原因**：本插件支持 `auto`（跟随 Obsidian），每次访问都需从 `settings.language` 实时解析。若用单例，每次设置变更都要记得调 `setLocale()`，容易遗漏导致翻译与设置不同步。让 `t()` 显式接收 locale，由 `plugin.locale` getter 解析，状态更直观、无隐式全局态。

```typescript
// main.ts
get locale(): Locale {
    return resolveLang(this.settings.language);
}

// 调用处
item.setTitle(t('menu.textColor', this.locale));
```

### 2.2 中文 locale 码：标准码 + 向后兼容

采用标准码 `zh-CN` / `zh-TW`（与 claudian、Obsidian 内部一致），而非原来的简码 `zh`。

**向后兼容**：老用户 `data.json` 里存的 `language: "zh"` 会在 `resolveLang` 中被映射为 `zh-CN`：

```typescript
export function resolveLang(setting: LangSetting): Locale {
    const raw = setting as string; // 磁盘读出的值运行时可能是任意字符串
    if (raw === 'auto') return detectObsidianLocale();
    if (raw === 'zh') return 'zh-CN'; // backward compat: legacy simplified-Chinese code
    return raw in translations ? (raw as Locale) : DEFAULT_LOCALE;
}
```

注意 `LangSetting` 类型已不含 `'zh'`，直接 `setting === 'zh'` 会触发 `TS2367`（类型无交集），故先断言为 `string` 再比对——`loadData()` 从磁盘读出的值在运行时可能是任意字符串，类型系统无法感知。下次用户改语言或保存设置时，值会被写回为 `zh-CN`，完成静默迁移。

### 2.3 `auto` 检测：从两档升级到 10 语言映射

原 `detectObsidianLang()` 只判断 `zh` 前缀和非空（非 zh 即 en）。新增 `matchLocale(raw)` 函数，把任意 locale 字符串映射到 10 种支持语言：

```typescript
export function matchLocale(raw: string): Locale {
    const lower = raw.toLowerCase();
    if (!lower) return DEFAULT_LOCALE;
    // 中文变体：zh-TW / zh-HK → 繁体，zh-CN / zh → 简体
    if (lower.startsWith('zh')) {
        return lower.includes('tw') || lower.includes('hk') || lower.includes('hant')
            ? 'zh-TW' : 'zh-CN';
    }
    if (lower.startsWith('ja')) return 'ja';
    if (lower.startsWith('ko')) return 'ko';
    if (lower.startsWith('de')) return 'de';
    if (lower.startsWith('fr')) return 'fr';
    if (lower.startsWith('es')) return 'es';
    if (lower.startsWith('ru')) return 'ru';
    if (lower.startsWith('pt')) return 'pt';
    return DEFAULT_LOCALE;
}
```

检测优先级：`activeWindow.moment.locale()`（Obsidian 始终同步 moment locale 为界面语言）→ `localStorage.getItem('language')` → 英文兜底。这样 Obsidian 设日语/韩语界面时，`auto` 也能正确显示对应语言，而非回退英文。

### 2.4 翻译键结构：dot-path 命名空间

JSON 采用两级嵌套命名空间，`colorNames` 作为 `colors.names` 子对象：

```json
{
  "menu": { "textColor": "Text color", "bold": "Bold", ... },
  "settingTab": { "general": "General", "colors": "Text Color", ... },
  "colors": {
    "heading": "Text color",
    "names": { "red": "Red", "orange": "Orange", ... }
  },
  "hyperlink": { "enabled": "Enable auto hyperlink", ... }
}
```

共 42 个键。消费方式：`t('menu.textColor', locale)`、`t('colors.names.red', locale)`。

### 2.5 编译期 key 校验

`TranslationKey` 从 `en.json` 递归推导为 dot-path 联合类型：

```typescript
import type * as en from './locales/en.json';

type DotJoin<Head extends string, Tail extends string> = `${Head}.${Tail}`;

type LeafKeys<T> = {
    [K in keyof T & string]: T[K] extends string
        ? K
        : T[K] extends Record<string, unknown>
            ? DotJoin<K, LeafKeys<T[K]>>
            : never;
}[keyof T & string];

export type TranslationKey = LeafKeys<typeof en>;
```

`t('menu.textColor', locale)` 合法；`t('menu.textcolor', locale)`（拼错）直接 TS 报错。需要 `resolveJsonModule: true`（tsconfig 已加）才能 `import type * as en from './locales/en.json'`。

## 4. 关键实现

### 4.1 `t()` 函数与回退逻辑（`i18n/i18n.ts`）

```typescript
const translations: Record<Locale, TranslationDict> = { en, 'zh-CN': zhCN, 'zh-TW': zhTW, ja, ko, de, fr, es, ru, pt };
const DEFAULT_LOCALE: Locale = 'en';

export function t(key: TranslationKey, locale: Locale, params?: Record<string, string | number>): string {
    const value = lookup(translations[locale], key);
    if (typeof value === 'string') return interpolate(value, params);

    // 缺键 → 回退英文
    if (locale !== DEFAULT_LOCALE) {
        const fallback = lookup(translations[DEFAULT_LOCALE], key);
        if (typeof fallback === 'string') return interpolate(fallback, params);
    }
    return key; // 仍缺 → 返回 key 字符串
}
```

`lookup` 沿 dot-path 逐层下钻；`interpolate` 用 `/\{(\w+)\}/g` 替换 `{param}` 占位符（当前无键使用占位符，但保留能力以备扩展）。

### 4.2 `getColorName()` —— 封装嵌套查找

`types.ts` 的 `getDefaultColors` / `colorsAreBuiltinDefaults` 需要按颜色 id 查名称。为避免在调用方拼 `colors.names.${id}` 字符串，封装为：

```typescript
export function getColorName(id: string, setting: LangSetting): string {
    const locale = resolveLang(setting);
    const key = `colors.names.${id}` as TranslationKey;
    return t(key, locale);
}
```

`colorsAreBuiltinDefaults` 用 `SUPPORTED_LOCALE_CODES.some((locale) => getColorName(def.id, locale as LangSetting) === color.name)` 检查名称是否匹配任一语言——这是「切换语言时是否安全重译调色板」的判断依据。

### 4.3 `constants.ts` —— 元数据驱动下拉框

```typescript
export const SUPPORTED_LOCALES: LocaleInfo[] = [
    { code: 'en', name: 'English', englishName: 'English', flag: '🇺🇸' },
    { code: 'zh-CN', name: '简体中文', englishName: 'Simplified Chinese', flag: '🇨🇳' },
    { code: 'zh-TW', name: '繁體中文', englishName: 'Traditional Chinese', flag: '🇹🇼' },
    // ja, ko, de, fr, es, ru, pt ...
];

export const LANGUAGE_OPTIONS: { value: LangSetting; label: string }[] = [
    { value: 'auto', label: 'Auto' },
    ...SUPPORTED_LOCALES.map((l) => ({ value: l.code as LangSetting, label: l.name })),
];
```

下拉框显示 endonym（简体中文 / 日本語 / Deutsch），无论当前 UI 语言为何都能识别自己的母语。新增语言只需在 `SUPPORTED_LOCALES` 追加一项 + 加一个 JSON 文件。

### 4.4 `main.ts` —— getter 从 `this.t` 改为 `this.locale`

```typescript
// 旧
get t(): Translations { return getTranslations(this.settings.language); }
// 菜单构建：const t = this.t; item.setTitle(t.menuTextColor);

// 新
get locale(): Locale { return resolveLang(this.settings.language); }
// 菜单构建：const locale = this.locale; item.setTitle(t('menu.textColor', locale));
```

`getDefaultColors` 签名从 `getDefaultColors(t: Translations)` 改为 `getDefaultColors(setting: LangSetting)`，内部用 `getColorName`。`DEFAULT_SETTINGS` 用 `getDefaultColors('en')`。

### 4.5 `settingTab.ts` —— 方法参数从 `t: Translations` 改为 `locale: Locale`

所有 `render*Tab(container, t)` 改为 `render*Tab(container, locale)`，方法体内 `t.settingLanguageName` → `t('language.name', locale)`。`display()` 顶部 `const t = this.plugin.t` → `const locale = this.plugin.locale`。语言切换回调中 `getDefaultColors(getTranslations(...))` → `getDefaultColors(this.plugin.settings.language)`。

## 6. 与 claudian 的对比

| 维度 | 本插件（text-brush） | claudian |
|------|---------------------|----------|
| locale 状态 | `t(key, locale)` 显式传参，无全局态 | 模块级 `currentLocale` 单例 + `setLocale()` |
| `auto` 检测 | ✅ 支持，`moment.locale()` → 10 语言映射 | ❌ 不支持，纯用户配置 |
| 回退 | 缺键 → 英文 → key 字符串 | 同 |
| 类型校验 | `TranslationKey` 从 en.json 推导 | 同 |
| JSON 拆分 | 10 个独立 JSON | 同（10 个） |
| locale 码 | `zh-CN`/`zh-TW` | `zh-CN`/`zh-TW` |

**未照搬 claudian 的点**：模块级单例 + `setLocale()`。因本插件有 `auto` 需求，显式传 locale 更不易出错。

## 8. 后续维护指引

### 新增一种语言（例如意大利语 `it`）

1. 复制 `locales/en.json` → `locales/it.json`，翻译所有 42 个键
2. `types.ts` 的 `Locale` 联合类型追加 `'it'`
3. `i18n.ts` 顶部 `import * as it from './locales/it.json'`，加入 `translations` 字典
4. `constants.ts` 的 `SUPPORTED_LOCALES` 追加 `{ code: 'it', name: 'Italiano', englishName: 'Italian', flag: '🇮🇹' }`
5. （可选）`matchLocale` 追加 `if (lower.startsWith('it')) return 'it';` 让 `auto` 识别

`LANGUAGE_OPTIONS` 下拉框会自动包含新语言（由 `SUPPORTED_LOCALES.map` 生成）。

### 新增一个翻译键

1. 在**所有 10 个** `locales/*.json` 中添加该键（先加 `en.json`，再同步其余）
2. `TranslationKey` 会自动包含新键（无需手改类型）
3. 运行 key 一致性校验脚本确认无遗漏

> ⚠️ 漏加某语言的键不会报错（运行时回退英文），但会导致该语言界面出现英文混入。建议每次加键后跑一遍一致性校验。
