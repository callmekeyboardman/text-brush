# 代码阅读指南

## 整体架构

本项目是一个 Obsidian 编辑器插件，采用**模块化架构**，源码位于 `src/` 目录下，按职责拆分为五个文件。插件通过监听编辑器右键菜单事件，向菜单注入颜色和字号选项，对选中文字执行 HTML `<span>` 包裹/替换/移除操作；同时监听粘贴事件，将纯 URL 自动转为 Markdown 链接。所有面向用户的文案（菜单项、设置标签、内置颜色名）集中在自包含的 i18n 层，支持 10 种语言切换。

> 💡 **设计要点 1：** `settingTab.ts` 对 `main.ts`、`hyperlinkPaste.ts` 对 `types.ts` 均使用 `import type`（虚线箭头 ⇢），编译后不产生运行时 `require()` 调用，从而避免循环依赖。
>
> 💡 **设计要点 2：** `i18n/` 目录刻意不依赖插件内任何其他文件（仅静态导入自带的 `locales/*.json`），因此 `types.ts` 可以安全地反向引用它（`types → i18n`）而不会形成循环。所有语言相关的数据和逻辑收敛于此单一模块。
>
> 💡 **设计要点 3：** `hyperlinkPaste.ts` 仅在类型层面引用 `HyperlinkSettings`，运行时逻辑（URL 解析、私有地址检测、HTML 标题抓取、编码解码）全部自包含，与颜色/字号模块完全解耦，互不影响。

## 程序入口

**入口文件：** `src/main.ts`（构建产物为根目录 `main.js`）

Obsidian 通过 `manifest.json` 中的 `"main": "main.js"` 定位插件入口，加载 `default export` 的 Plugin 子类。插件生命周期由 `onload()` 方法启动。

## 源码文件简介

### src/main.ts — 插件主逻辑

继承 `Plugin` 的入口类 `TextColorPlugin`，核心工作有二：其一，监听 `editor-menu` 事件，通过 `resolveTarget()` 智能解析选区，构建两个独立菜单（文字颜色、文字大小），执行着色/清除/加粗/字号调整操作；其二，在 `onload` 中实例化 `HyperlinkPasteHandler` 并注册 `editor-paste` 监听，将粘贴 URL 转链接的逻辑委托给它，`onunload` 时调用 `destroy()` 中止异步回调，`saveSettings` 时调用 `updateSettings()` 使设置实时生效。

> 详见 [知识卡片-1-main.ts.md](知识卡片-1-main.ts.md)

### src/types.ts — 数据模型层

定义插件所有共享的数据结构（`ColorOption`、`FontSizeOption`、`TextColorSettings`、`ResolvedTarget`、`HyperlinkSettings`）、默认调色板定义（`DEFAULT_COLOR_DEFS`）和字号常量、用于检测/解析 span 标签的正则表达式，以及颜色、加粗、字号三类工具函数。内置颜色名称不再硬编码，而是通过 `getDefaultColors(setting)` 调用 `getColorName()` 从当前语言的翻译表生成；`colorsAreBuiltinDefaults()` 用于判断调色板是否仍为未修改的默认值。`HyperlinkSettings` 接口与 `DEFAULT_SETTINGS.hyperlink` 为粘贴超链接功能提供数据结构；`SettingsTabId` 同步增加 `'hyperlink'`。

> 详见 [知识卡片-2-types.ts.md](知识卡片-2-types.ts.md)

### src/i18n/ — 国际化层

轻量 i18n 模块（不 import 插件内任何其他文件，仅静态导入自带的 `locales/*.json`）。集中存放所有面向用户的文案：右键菜单项、设置页标签、内置颜色名、超链接设置页文案，共 42 个翻译键 × 10 种语言（en、zh-CN、zh-TW、ja、ko、de、fr、es、ru、pt）。目录拆为三个 TS 文件 + `locales/` JSON 目录：

- `i18n.ts` — 导出 `t(key, locale, params?)` 翻译函数（dot-path 访问，缺键自动回退英文，仍缺返回 key 字符串）、`resolveLang()`（将用户设置含 `auto` 解析为具体 locale）、`matchLocale()`（把任意 locale 字符串映射到支持的语言）、`getColorName()`（封装 `colors.names.{id}` 查找）、`getTranslations()`（返回完整 dict，供 `getDefaultColors` 用）。
- `types.ts` — `Locale` 联合类型、`LangSetting`（含 `auto`）、`TranslationKey`（从 `en.json` 递归推导的 dot-path 联合类型，编译期校验 key）、`TranslationDict`。
- `constants.ts` — `SUPPORTED_LOCALES` 元数据（code/name/englishName/flag）、`LANGUAGE_OPTIONS` 下拉选项、`getLocaleDisplayName()`。
- `locales/*.json` — 每种语言一个 JSON，`en.json` 为源真理，其余 9 个与之结构对齐。

`auto` 通过 `activeWindow.moment.locale()` 读取 Obsidian 界面语言（读不到回退 `localStorage`），经 `matchLocale` 映射到 10 种语言之一，无法识别时回退英文。

> 详见 [知识卡片-7-多语言架构.md](知识卡片-7-多语言架构.md)

### src/hyperlinkPaste.ts — 粘贴自动超链接

不依赖任何插件内部模块（仅 `import type` 引用 `HyperlinkSettings`），从独立插件 `obsidian-hyperlink` 迁移而来。导出 `HyperlinkPasteHandler` 类，监听粘贴事件：选中文本时立即生成 `[选中文本](url)`；无选区时插入占位符，异步 `fetchTitle` 抓取页面标题后替换，失败回退 `[url](url)`。包含 URL 解析、私有/本地地址检测、排除域名（含子域与通配符）匹配、HTML 标题提取（DOMParser 优先、正则兜底）、HTML 实体解码、非 UTF-8 编码（含 GBK/GB2312）解码等纯工具函数。通过 `destroy()` 标志位保证插件卸载后异步回调不再触碰编辑器。

> 详见 [知识卡片-6-URL转链接技术方案.md](知识卡片-6-URL转链接技术方案.md)

### src/settingTab.ts — 设置页 UI

继承 `PluginSettingTab` 的设置页类，采用 Tab 导航结构（通用 / 文字颜色 / 字体 / 超链接）。通用 Tab 提供语言选择下拉（Auto + 10 种语言），文字颜色 Tab 和字体 Tab 提供列表的实时编辑（名称/值/删除）、新增自定义选项、一键恢复默认功能；超链接 Tab 提供启用开关、抓取超时、跳过私有地址、User-Agent、排除域名等配置。所有文案通过 `t(key, locale)` 函数调用获取，`locale` 由 `plugin.locale` getter 解析当前设置。所有修改即时保存无需确认。当前激活的 Tab 持久化到设置中，重新打开设置页时自动恢复。

> 详见 [知识卡片-3-settingTab.ts.md](知识卡片-3-settingTab.ts.md)

### styles.css — 样式定义

为 9 种预设颜色提供 `.tc-color--{id}` 修饰类（使用 Obsidian 主题变量），以及菜单/设置页中 12×12 色块预览的 `.tc-menu-swatch` 样式。用户可通过 CSS Snippet 用 `!important` 覆盖 inline style。

### versions.json — 版本兼容映射

格式为 `"插件版本": "最低 Obsidian 版本"`，Obsidian 客户端据此判断用户能安装的最高兼容版本。发布新版且使用了新 API 时需追加映射。

## 关键设计决策

1. **模块化拆分** — 按职责分为数据模型（types）、业务逻辑（main）、UI（settingTab），便于维护和扩展
2. **`import type` 避免循环依赖** — settingTab 仅在类型层面引用 main，运行时无循环
3. **双输出（class + inline style）** — 兼顾主题定制性和导出场景的可靠性
4. **稳定 ID 而非颜色值作 class 名** — 用户修改颜色值后，已标记文字的 class 不变，CSS Snippet 仍生效
5. **运行时探测子菜单** — 不依赖特定 Obsidian 版本号，而是直接检测 API 存在性
6. **选区智能扩展** — 解决 Live Preview 模式下用户无法选中 HTML 标签的问题，避免嵌套 span。分多层兜底：紧邻标签用 OPEN/CLOSE_TAG_RE 扩展、span 内部用 SPAN_IN_LINE_RE 定位、**与 span 部分重叠（含跨行）用 `expandToOverlappingSpan()` 扩边到整个 span**，确保绝不留下残缺标签
7. **XSS 防护** — `escapeAttr()` 转义用户自定义颜色值中的特殊字符
8. **自包含 i18n 层** — 所有文案集中于 `i18n/` 目录（10 种语言 JSON + `t(key, locale)` 函数），且该模块零内部依赖，可被各层自由引用；`TranslationKey` 从 `en.json` 推导实现编译期 key 校验；缺键自动回退英文；语言解析支持 `auto` 跟随 Obsidian（10 种语言映射）；切换语言时仅在调色板仍为默认时重译，绝不覆盖用户自定义
9. **设置页 Tab 化 + 状态持久化** — 设置页按功能域拆为通用/文字颜色/字体/超链接四个 Tab，新增功能只需追加 Tab ID 和渲染方法；当前激活的 Tab 存入 `settings.activeSettingsTab`，重新打开设置页时自动恢复
10. **粘贴处理器隔离** — `HyperlinkPasteHandler` 封装为独立类，所有 URL/网络逻辑收敛于 `hyperlinkPaste.ts`，仅通过 `import type` 依赖 `HyperlinkSettings`，与颜色/字号模块零运行时耦合；`destroy()` 标志位确保插件卸载后异步标题抓取回调不再修改编辑器，`updateSettings()` 使设置变更无需重启即时生效；`editor-paste` 与 `editor-menu` 两个事件监听相互独立，互不干扰
