# 代码阅读指南

## 整体架构

本项目是一个 Obsidian 编辑器插件，采用**模块化架构**，源码位于 `src/` 目录下，按职责拆分为四个文件。插件通过监听编辑器右键菜单事件，向菜单注入颜色和字号选项，对选中文字执行 HTML `<span>` 包裹/替换/移除操作。所有面向用户的文案（菜单项、设置标签、内置颜色名）集中在自包含的 i18n 层，支持中文/英文切换。

┌─────────────────────────────────────────────────────┐
│                    Obsidian App                      │
│                                                     │

│  │ Editor (CM6)  │────▶│ editor-menu 事件     │      │
│  └───────────────┘     └────────┬────────────┘      │

### 模块依赖关系

```
                    ┌─────────────────────────────────────┐
                    │           src/main.ts               │
                    │       (插件主类 · 入口文件)           │
                    └───────────┬───────────┬─────────────┘
                                │           │
                  imports       │           │  imports
            (类型+常量+函数)     │           │  (设置页类)
                                │           │
                    ┌───────────▼──┐   ┌────▼──────────────┐
                    │ src/types.ts │   │ src/settingTab.ts  │
                    │ (数据模型层)  │   │ (设置页 UI)        │
                    └───────────▲──┘   └──┬────────────────┘
                                │         │
                  imports       │         │  import type
            (常量 DEFAULT_COLORS)│         │  (仅类型引用，无运行时依赖)
                                │         │
                                └─────────┘
```

**依赖清单：**

| 来源 | 方向 | 目标 | 引入内容 |
|------|------|------|----------|
| `main.ts` | → | `types.ts` | `ColorOption`, `FontSizeOption`, `TextColorSettings`, `ResolvedTarget`, 常量, 正则, 工具函数 |
| `main.ts` | → | `settingTab.ts` | `TextColorSettingTab` 类 |
| `main.ts` | → | `i18n.ts` | `Translations` 类型, `getTranslations()` |
| `settingTab.ts` | → | `types.ts` | `getDefaultColors`, `colorsAreBuiltinDefaults` |
| `settingTab.ts` | → | `i18n.ts` | `LangSetting` 类型, `getTranslations`, `LANGUAGE_OPTIONS` |
| `settingTab.ts` | ⇢ (type-only) | `main.ts` | `TextColorPlugin` 类型签名 |
| `types.ts` | → | `i18n.ts` | `LangSetting`/`Translations` 类型, `SUPPORTED_LANGS`, `TRANSLATIONS` |
| `i18n.ts` | — | （无） | 自包含，不 import 任何插件内部模块 |

> 💡 **设计要点 1：** `settingTab.ts` 对 `main.ts` 使用 `import type`（虚线箭头 ⇢），编译后不产生运行时 `require()` 调用，从而避免 main ↔ settingTab 的循环依赖。
>
> 💡 **设计要点 2：** `i18n.ts` 刻意不依赖插件内任何其他文件（自包含），因此 `types.ts` 可以安全地反向引用它（`types → i18n`）而不会形成循环。所有语言相关的数据和逻辑收敛于此单一模块。

## 程序入口

**入口文件：** `src/main.ts`（构建产物为根目录 `main.js`）

Obsidian 通过 `manifest.json` 中的 `"main": "main.js"` 定位插件入口，加载 `default export` 的 Plugin 子类。插件生命周期由 `onload()` 方法启动。

## 源码文件简介

### src/main.ts — 插件主逻辑

继承 `Plugin` 的入口类 `TextColorPlugin`，核心工作是监听 `editor-menu` 事件，通过 `resolveTarget()` 智能解析选区，然后构建两个独立菜单（文字颜色、文字大小），执行着色/清除/加粗/字号调整操作。

> 详见 [知识卡片-1-main.ts.md](知识卡片-1-main.ts.md)

### src/types.ts — 数据模型层

定义插件所有共享的数据结构（`ColorOption`、`FontSizeOption`、`TextColorSettings`、`ResolvedTarget`）、默认调色板定义（`DEFAULT_COLOR_DEFS`）和字号常量、用于检测/解析 span 标签的正则表达式，以及颜色、加粗、字号三类工具函数。内置颜色名称不再硬编码，而是通过 `getDefaultColors(t)` 从当前语言的翻译表生成；`colorsAreBuiltinDefaults()` 用于判断调色板是否仍为未修改的默认值。

> 详见 [知识卡片-2-types.ts.md](知识卡片-2-types.ts.md)

### src/i18n.ts — 国际化层

自包含的轻量 i18n 模块（不 import 插件内任何其他文件）。集中存放所有面向用户的文案：右键菜单项、设置页标签、内置颜色名。导出 `Lang`/`LangSetting` 类型、`TRANSLATIONS` 翻译表（`en`/`zh`）、`LANGUAGE_OPTIONS` 下拉选项，以及 `resolveLang()`/`getTranslations()` —— 后者将用户设置（含 `auto`）解析为具体语言；`auto` 通过 `activeWindow.localStorage` 读取 Obsidian 界面语言，无法识别时回退英文。

### src/settingTab.ts — 设置页 UI

继承 `PluginSettingTab` 的设置页类，顶部提供语言选择下拉（Auto / English / 中文），下方提供颜色列表和字号列表的实时编辑（名称/值/删除）、新增自定义选项、一键恢复默认功能，所有修改即时保存无需确认。

> 详见 [知识卡片-3-settingTab.ts.md](知识卡片-3-settingTab.ts.md)

### styles.css — 样式定义

为 9 种预设颜色提供 `.tc-color--{id}` 修饰类（使用 Obsidian 主题变量），以及菜单/设置页中 12×12 色块预览的 `.tc-menu-swatch` 样式。用户可通过 CSS Snippet 用 `!important` 覆盖 inline style。

### versions.json — 版本兼容映射

格式为 `"插件版本": "最低 Obsidian 版本"`，Obsidian 客户端据此判断用户能安装的最高兼容版本。发布新版且使用了新 API 时需追加映射。

## 关键设计决策

1. **模块化拆分** — 按职责分为数据模型（types）、业务逻辑（main）、UI（settingTab），便于维护和扩展
2. **`import type` 避免循环依赖** — settingTab 仅在类型层面引用 main，运行时无循环
3. **双输出（class + inline style）** — 兼顾主题定制性和导出场景的可靠性
4. **稳定 ID 而非颜色值作 class 名** — 用户修改颜色值后，已标记文字的 class 不变，CSS Snippet 仍生效
5. **运行时探测子菜单** — 不依赖特定 Obsidian 版本号，而是直接检测 API 存在性
6. **选区智能扩展** — 解决 Live Preview 模式下用户无法选中 HTML 标签的问题，避免嵌套 span
7. **XSS 防护** — `escapeAttr()` 转义用户自定义颜色值中的特殊字符
8. **自包含 i18n 层** — 所有文案集中于 `i18n.ts`，且该模块零内部依赖，可被各层自由引用；语言解析支持 `auto` 跟随 Obsidian；切换语言时仅在调色板仍为默认时重译，绝不覆盖用户自定义
