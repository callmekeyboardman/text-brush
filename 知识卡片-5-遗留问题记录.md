# 遗留问题记录

代码 review 中发现的问题清单。问题 1、2 已修复，问题 3、4 暂缓处理，问题 2 的多 span 重叠边界限制也在此记录。

---

## 已修复

### 问题 1：reselect 多行末列计算错误

- **位置**：`src/main.ts` `reselect()`
- **现象**：替换文本跨多行且 `from.ch !== 0` 时，末行列号 `endCh = lines[末行].length` 漏加 `from.ch`，导致替换后选区末尾停在错误位置（偏左）。
- **修复**：多行分支改为 `from.ch + lines[lines.length - 1].length`。

### 问题 2：跨 span 边界选区残留标签

- **位置**：`src/main.ts` `resolveTarget()` + `applyColor()`
- **现象**：选区从 span 外选到 span 内（或反向）时，`wrapped=false`，`applyColor` 的 `stripColorSpans` 只清理完整 span，对残片（半个 open/close 标签）原样保留，`wrap` 后产生未闭合/嵌套错乱的 span。
- **修复**：`resolveTarget` 在 fallback 前新增"部分重叠扩边"分支，把替换范围扩展到覆盖整个相交 span。新增 `expandToOverlappingSpan()` 和 `offsetToPos()` 两个私有方法，`src/types.ts` 新增 `SPAN_IN_BLOCK_RE` 正则。

## 暂缓处理

### 问题 3：`addBoldToStyle` / `addFontSizeToStyle` 依赖 style 属性必须存在

- **位置**：`src/types.ts:150-152`、`src/types.ts:178-183`
- **代码**：

```typescript
export function addBoldToStyle(spanHtml: string): string {
    return spanHtml.replace(/(style="[^"]*)/, '$1; font-weight: bold');
}
```

- **问题**：正则 `(style="[^"]*)` 对**没有 style 属性的 span 静默失败**——匹配不到时返回原串，加粗/字号被无声丢弃。
- **当前影响**：这两个函数只在 `target.wrapped === true`（即 current 是本插件产生的、带 `style=` 的 span）时调用，调用链保证了前提成立，**当前无实际 bug**。
- **为何暂缓**：是一个隐含耦合，而非现存缺陷。只有未来出现"无 style 的 tc-color span"进入这条路径时才会触发。
- **后续建议**：加防御性兜底——匹配不到 `style="` 时，注入 `style="font-weight: bold"`（或对应字号）而非原样返回。可加单测固化"无 style span 也能加粗"。

### 问题 4：`color.value` 直接当 CSS 值使用，无校验反馈

- **位置**：
  - `src/types.ts:189-197` `createColorTitle()` —— `swatch.style.backgroundColor = color.value`
  - `src/settingTab.ts:98`、`src/settingTab.ts:117` —— 设置页色块同样直接赋值
- **问题**：用户在设置里可填任意字符串作为颜色值。无效值（如 `notacolor`）或未知变量（如 `var(--undefined-var)`）会让色块显示为透明/默认色，**不给出任何反馈**，用户难以发现配置错误。
- **当前影响**：纯体验问题。因为是写入 `style` 属性而非 `innerHTML`，**无 XSS 风险**，浏览器对无效 CSS 值会静默忽略。
- **为何暂缓**：不影响数据正确性和安全性，仅影响配置时的反馈体验。
- **后续建议**：
  - 在设置页颜色值输入框旁加一个实时预览，无效值时显示警告样式（如色块打叉、输入框红框）。
  - 或在 `onChange` 时用 DOM 探测法校验：临时赋值后比较 `element.style.backgroundColor` 是否为空字符串，为空则提示无效。

## 已知边界限制

### 问题 2 的多 span 重叠场景

- **位置**：`src/main.ts` `expandToOverlappingSpan()`
- **限制**：当选区**同时跨越多个 span**（例如选中 `<span>A</span> 文本 <span>B</span>` 中从 A 内部到 B 内部的大段内容）时，`expandToOverlappingSpan` 只会扩边到**第一个相交的 span**。被跨越的后续 span 和中间文本仍可能留下残片标签。
- **当前缓解**：`applyColor` 里的 `stripColorSpans` 会清理选区内所有**完整**的 span，所以中间完整的 span 不会残留；只有"被选区边缘截断的最后一个 span"可能残留半个标签。
- **为何暂不处理**：这是极端场景（用户手动选了跨越多个 span + 中间文本的大段内容），实际使用中罕见。完整处理需要把替换范围扩展到覆盖所有相交 span 的并集，逻辑显著复杂化。
- **后续建议**：若要彻底解决，`expandToOverlappingSpan` 应返回**所有相交 span 的最小外接区间**（从最早相交 span 的起点到最后相交 span 的终点），并保证 `inner` 不含半个标签。可结合 `stripColorSpans` 对扩边后范围内的文本做二次清理。
